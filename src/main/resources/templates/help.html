SysStatz - Hilfe & Dokumentation

Übersicht

Willkommen in der SysStatz-Hilfe. Hier finden Sie Anleitungen, Tipps und Erklärungen zu allen Funktionen der Software.

Server Übersicht

In der Serverübersicht können Sie:

  • Serverstatus sehen
  • CPU, RAM, Netzwerk und Speicher überwachen
  • Server stoppen oder neu starten (wenn aktiviert)
  • Serverdetails und Vergleiche aufrufen

PDF-Auswertung

Mit der PDF-Auswertung können Sie:

  • Ausgewählte Server und Metriken auswählen
  • PDF-Berichte erstellen
  • Berichte auf dem Server speichern oder herunterladen

Server vergleichen

Mit dem Vergleichs-Tool können Sie mehrere Server nebeneinander darstellen und deren Statistiken analysieren.

Suchfunktion

Die Suchleiste oben erlaubt es, Server schnell nach Name zu filtern.

Häufige Probleme

  • Keine Serverdaten verfügbar: Stellen Sie sicher, dass die Server verbunden sind.
  • PDF wird nicht heruntergeladen: Prüfen Sie, ob die Datei auf dem Server vorhanden ist.

Support & Kontakt

Bei fragen schreiben sie gerne eine E-Mail an support@sysstatz.de.

Server mit dem Dashboard verbinden

  1. Installieren Sie den SysStatz Client auf dem Zielserver.
  2. Geben Sie während der Installation die IP des Statsservers ein. Dies ist die öffentliche IP des Servers, auf dem der Statsserver läuft.
  3. Geben Sie den Port des Statsservers an. Diesen finden Sie in den Einstellungen des Dashboards oder können ihn bei Bedarf ändern.
  4. Ermitteln Sie die IP-Adresse des Zielservers (Client), die vom Statsserver gesehen wird:
    • Windows (lokales Netzwerk): Öffnen Sie die Konsole (CMD) und geben Sie ipconfig ein. Merken Sie sich die IPv4-Adresse.
    • Linux (lokales Netzwerk): Öffnen Sie das Terminal und geben Sie ifconfig oder ip a ein (unter Umständen muss das Tool installiert werden). Merken Sie sich die IPv4-Adresse.
    • Windows/Linux (Internet / öffentliche IP): Öffnen Sie einen Browser oder das Terminal und prüfen Sie die öffentliche IP:
      • Web: whatismyipaddress.com
      • Linux Terminal: curl ifconfig.me oder curl icanhazip.com
      • Windows Terminal (nur PowerShell): (Invoke-WebRequest ifconfig.me/ip).Content.Trim()
  5. Tragen Sie die ermittelte IP im Dashboard unter „Server hinzufügen“ ein.

Tipps, falls der Server nicht erscheint:

  • Prüfen Sie die Logs des Statsservers:
  • Prüfen Sie die Logs des Clients.
  • Wenn das Problem weiterhin besteht, kontaktieren Sie bitte den Support oder den Administrator.

Hinweis: Die öffentliche IP ist die Adresse, unter der der Statsserver im Internet erreichbar ist. Die private IP (z. B. 192.168.x.x) funktioniert nur im lokalen Netzwerk. Achten Sie darauf, die richtige IP zu verwenden:

  • Lokales Netzwerk → private IP (ipconfig / ifconfig)
  • Von außerhalb (Internet) → öffentliche IP (Webdienste oder curl)

Langzeitanalyse

Mit der Langzeitanalyse können sie:

  • Die Auslastung von CPU, RAM und Festplatte über den ganzen Zeitraum sehen, seitdem der Server verbunden ist.
  • Spikes erkennen
  • Duchgehendes Ansteigen der Auslastung erkennen

KI-Analyse

Mit der KI-Analyse können sie:

  • Serverdaten von KI auswerten lassen
  • Verbesserungsvorschläge bekommen.
